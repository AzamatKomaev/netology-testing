# Тест-кейсы - F-Bank

Все шаги выполняются вручную в Google Chrome. Перед каждым тест-кейсом приложение открывается с указанным в кейсе URL, в строке нет лишних параметров, кеш браузера очищен (или используется приватное окно).

---

## TC-01. Успешный перевод в рублях

| Поле | Значение |
|------|----------|
| **ID** | TC-01 |
| **Заголовок** | Перевод корректной суммы в рублях с достаточным остатком на счёте |
| **Приоритет** | High |
| **Тип** | Позитивный |

**Предусловия:**
- Приложение открыто по адресу `http://localhost:8000/?balance=10000&reserved=0`.

**Шаги:**
1. Кликнуть по карточке "Рубли".
2. В поле "Номер карты:" ввести `1234567812345678`.
3. В поле "Сумма перевода:" ввести `1000`.
4. Убедиться, что отображается строка "Комиссия: 100 rub".
5. Нажать кнопку "Перевести".

**Ожидаемый результат:**
- Появляется браузерный alert с текстом `Перевод 1000 rub на карту 1234567812345678 принят банком!`.
- Сообщение "Недостаточно средств на счете" не отображается.

**Фактический результат:** соответствует ожидаемому.

**Статус:** Pass

## TC-02. Сообщение о недостатке средств

| Поле | Значение |
|------|----------|
| **ID** | TC-02 |
| **Заголовок** | Кнопка "Перевести" скрыта, отображается ошибка при недостатке средств |
| **Приоритет** | High |
| **Тип** | Позитивный (негативный сценарий с точки зрения пользователя) |

**Предусловия:**
- Приложение открыто по адресу `http://localhost:8000/?balance=500&reserved=0`.

**Шаги:**
1. Кликнуть по карточке "Рубли".
2. В поле "Номер карты:" ввести `1234567812345678`.
3. В поле "Сумма перевода:" ввести `1000`.

- Отображается красная надпись `Недостаточно средств на счете`.

**Фактический результат:** соответствует ожидаемому.

**Статус:** Pass

## TC-03. Длина номера карты ограничена 16 цифрами

| Поле | Значение |
|------|----------|
| **ID** | TC-03 |
| **Заголовок** | Поле "Номер карты:" не должно принимать более 16 цифр |
| **Приоритет** | Medium |
| **Тип** | Негативный |
| **Связанный баг-репорт** | [BUG-002](bug-reports/BUG-002-card-17-digits.md) |

**Предусловия:**
- Приложение открыто по адресу `http://localhost:8000/?balance=10000&reserved=0`.

**Шаги:**
1. Кликнуть по карточке "Рубли".
2. В поле "Номер карты:" ввести подряд 17 цифр: `12345678123456789`.
3. Посмотреть, какая строка осталась в поле ввода.

**Ожидаемый результат:**
- В поле остаётся ровно 16 цифр (отображаются как `1234 5678 1234 5678`), 17-я цифра отбрасывается.
- Ниже появляется поле "Сумма перевода:".

**Фактический результат:**
- В поле остаётся 17 цифр (отображаются как `1234 5678 1234 5678 9`).
- Поле "Сумма перевода:" появляется уже после 16-й цифры, но дальнейший ввод не пресекается, и в alert уходит 17-значный "номер карты".

**Статус:** Fail (см. BUG-002)

## TC-04. Запрет отрицательной суммы перевода

| Поле | Значение |
|------|----------|
| **ID** | TC-04 |
| **Заголовок** | Поле "Сумма перевода:" не должно принимать отрицательные значения |
| **Приоритет** | Critical |
| **Тип** | Негативный |
| **Связанный баг-репорт** | [BUG-001](bug-reports/BUG-001-negative-amount.md) |

**Предусловия:**
- Приложение открыто по адресу `http://localhost:8000/?balance=10000&reserved=0`.

**Шаги:**
1. Кликнуть по карточке "Рубли".
2. В поле "Номер карты:" ввести `1234567812345678`.
3. В поле "Сумма перевода:" ввести `-100`.
4. Посмотреть на значение поля и состояние кнопки "Перевести".
5. Нажать "Перевести" (если кнопка отображается).

**Ожидаемый результат:**
- Поле либо отбрасывает знак минус, либо сумма приводится к 0.
- Кнопка "Перевести" неактивна (или выводится сообщение об ошибке валидации).
- Перевод с отрицательной суммой невозможен.

**Фактический результат:**
- Знак минус сохраняется, в поле остаётся `-100`.
- Комиссия пересчитывается в `-10 rub` (отрицательная).
- Кнопка "Перевести" отображается и активна.
- При нажатии появляется alert: `Перевод -100 rub на карту 1234567812345678 принят банком!`

**Статус:** Fail (см. BUG-001)

## TC-05. Проверка остатка по выбранной валюте

| Поле | Значение |
|------|----------|
| **ID** | TC-05 |
| **Заголовок** | Сообщение "Недостаточно средств" должно учитывать остаток на выбранной валютной карточке |
| **Приоритет** | High |
| **Тип** | Негативный |
| **Связанный баг-репорт** | [BUG-003](bug-reports/BUG-003-currency-mismatch.md) |

**Предусловия:**
- Приложение открыто по адресу `http://localhost:8000/?balance=0&reserved=0` (рублёвый счёт пуст; долларовая карта по умолчанию имеет 100 usd).

**Шаги:**
1. Кликнуть по карточке "Доллары" (отображает "На счету: 100 usd").
2. В поле "Номер карты:" ввести `1234567812345678`.
3. В поле "Сумма перевода:" ввести `50` (меньше остатка USD).

**Ожидаемый результат:**
- Кнопка "Перевести" отображается, сообщения "Недостаточно средств" нет - у пользователя достаточно долларов.

**Фактический результат:**
- Кнопка "Перевести" **не отображается**, выводится `Недостаточно средств на счете`, хотя на долларовом счёте 100 usd.
- Приложение проверяет остаток рублёвого счёта (0 rub) вне зависимости от того, какая карточка выбрана.

**Статус:** Fail (см. BUG-003)
