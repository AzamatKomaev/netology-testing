# BUG-003 - Проверка достаточности средств всегда использует рублёвый счёт

| Поле | Значение |
|------|----------|
| **ID** | BUG-003 |
| **Заголовок** | При переводе с долларового или евро счёта приложение проверяет остаток рублёвого счёта |
| **Severity (серьёзность)** | Critical |
| **Priority (приоритет)** | P1 |
| **Окружение** | macOS 14.6 / Google Chrome 134; `python3 -m http.server 8000` |
| **Версия / билд** | `dist/assets/index-BUH56GOL.js` |
| **Автор** | QA |
| **Связанный тест-кейс** | [TC-05](../test-cases.md#tc-05-проверка-остатка-по-выбранной-валюте) |
| **Автотест** | [tests/test_currency_balance.py](../../automated/tests/test_currency_balance.py) |

## Описание

В приложении предусмотрены три счёта: рубли (баланс задаётся URL-параметром `balance`), доллары (фиксированный баланс 100 usd) и евро (фиксированный баланс 300 eur). Условие отображения кнопки "Перевести" вычисляется по формуле

```
balance_rub - reserved_rub - commission - amount > 0
```

и не учитывает, какой счёт пользователь выбрал. В результате:

1. При пустом рублёвом счёте (`balance=0`) **любой** перевод с долларовой или евро карточки отклоняется как "Недостаточно средств", даже если сумма меньше валютного остатка.
2. При большом рублёвом счёте (`balance=1000000`) пользователь может "перевести" сумму в долларах или евро, многократно превышающую валютный остаток (например, 10 000 usd при балансе 100 usd).

Оба поведения недопустимы: первое блокирует валидные переводы, второе позволяет обходить лимит счёта.

## Шаги воспроизведения (сценарий А - ложный отказ)

1. Открыть `http://localhost:8000/?balance=0&reserved=0`.
2. Кликнуть по карточке "Доллары" (на ней отображается "На счету: 100 usd").
3. В поле "Номер карты:" ввести `1234 5678 1234 5678`.
4. В поле "Сумма перевода:" ввести `50` (заведомо меньше остатка USD).

## Ожидаемый результат (сценарий А)

- Кнопка "Перевести" отображается.
- Сообщение "Недостаточно средств на счете" отсутствует.

## Фактический результат (сценарий А)

- Отображается красное `Недостаточно средств на счете`.
- Кнопка "Перевести" скрыта.

## Шаги воспроизведения (сценарий Б - обход лимита)

1. Открыть `http://localhost:8000/?balance=1000000&reserved=0`.
2. Кликнуть по карточке "Доллары".
3. Ввести номер карты `1234 5678 1234 5678` и сумму `1000` (при остатке 100 usd).
4. Нажать "Перевести".

## Ожидаемый результат (сценарий Б)

- Сообщение "Недостаточно средств на счете", кнопка скрыта.

## Фактический результат (сценарий Б)

- Перевод проходит, отображается alert: `Перевод 1000 usd на карту 1234567812345678 принят банком!`

## Анализ кода

```js
const o = Number(i.get("balance") || "0");
const s = Number(i.get("reserved") || "0");
// ...
const U = [
  { title: "Рубли",   label: "rub",  balance: o,   reserved: s,  currency: "rub" },
  { title: "Доллары", label: "usd",  balance: 100, reserved: 0,  currency: "usd" },
  { title: "Евро",    label: "euro", balance: 300, reserved: 26, currency: "eur" }
];
// ...
const A = o - s - h - O > 0;            // <-- всегда RUB-остаток
```

Переменные `o` и `s` зашиты в условие напрямую вместо использования полей выбранной карточки `U.find(card => card.label === f)`.

## Рекомендация

Брать баланс и резерв из выбранной валютной карточки:

```js
const selected = U.find(card => card.label === f);
const A = selected
    ? selected.balance - selected.reserved - h - O > 0
    : false;
```

При появлении бэкенда - выполнять проверку платёжеспособности на сервере, поскольку клиентскую можно обойти модификацией состояния.
