# BUG-001 - Поле "Сумма перевода" принимает отрицательные значения

| Поле | Значение |
|------|----------|
| **ID** | BUG-001 |
| **Заголовок** | F-Bank принимает к переводу отрицательную сумму, что приводит к зачислению средств на счёт получателя за минусовое списание у отправителя |
| **Severity (серьёзность)** | Critical |
| **Priority (приоритет)** | P1 - must fix |
| **Окружение** | macOS 14.6 / Google Chrome 134; приложение `dist/`, обслуживаемое через `python3 -m http.server 8000` |

| **Автор** | QA |

## Описание

Поле "Сумма перевода:" при вводе значения с ведущим знаком `-` сохраняет его и интерпретирует число как отрицательное. Внутренняя логика перевода не валидирует знак и:

- пересчитывает комиссию по формуле `floor(amount / 100) * 10`, давая отрицательную комиссию;
- считает пользователя платёжеспособным, так как условие `balance - reserved - commission - amount > 0` при отрицательной `amount` всегда смещается в плюс;
- разрешает нажать кнопку "Перевести" и подтверждает "успешный" перевод отрицательной суммы.

В реальном банке подобное поведение позволило бы злоумышленнику списать деньги со счёта получателя в свою пользу.

## Шаги воспроизведения

1. Открыть `http://localhost:8000/?balance=10000&reserved=0`.
2. Кликнуть по карточке "Рубли".
3. В поле "Номер карты:" ввести `1234 5678 1234 5678`.
4. В поле "Сумма перевода:" очистить значение и ввести `-100`.
5. Нажать кнопку "Перевести".

## Ожидаемый результат

- Поле либо игнорирует ведущий минус (превращая ввод в положительное число), либо приводит сумму к `0`.
- Кнопка "Перевести" неактивна (или выводится сообщение валидации).
- При попытке отправить отрицательную сумму перевод не выполняется.

## Фактический результат

- В поле остаётся значение `-100`.
- Под полем суммы отображается строка `Комиссия: -10 rub`.
- Кнопка "Перевести" отображается и кликабельна.
- После нажатия появляется системный alert:
  `Перевод -100 rub на карту 1234567812345678 принят банком!`

## Анализ кода

В `dist/assets/index-BUH56GOL.js` (минифицированный сборочный артефакт) обработчик ввода суммы перевода:

```js
onChange: R => {
    let L = R.target.value.startsWith("-"),
        j = R.target.value.replace(/\D/g, "");
    L && (j = "-" + j);                 // <-- знак "-" восстанавливается
    R.target.value = j;
    (j === "" || j === "-") && (j = "0");
    p(Number(j));                       // <-- состояние amount получает отрицательное число
}
```

Условие отображения кнопки:

```js
const A = o - s - h - O > 0;            // o=balance, s=reserved, h=commission, O=amount
```

При `O < 0` выражение становится больше, чем `o - s`, поэтому `A` истинно, и кнопка перевода всегда видна.

## Рекомендация

В обработчике `onChange` суммы перевода удалить ветку, восстанавливающую знак минус:

```js
let j = R.target.value.replace(/\D/g, ""); // только цифры
R.target.value = j;
```

Дополнительно - добавить проверку `O > 0` в условие отображения кнопки и/или серверную валидацию знака суммы.
