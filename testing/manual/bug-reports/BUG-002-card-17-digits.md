| Поле | Значение |
|------|----------|
| **ID** | BUG-002 |
| **Заголовок** | Поле "Номер карты:" принимает 17 цифр вместо стандартных 16 |
| **Severity (серьёзность)** | Major |
| **Priority (приоритет)** | P2 |
| **Окружение** | macOS 14.6 / Google Chrome 134; `python3 -m http.server 8000` |
| **Версия / билд** | `dist/assets/index-BUH56GOL.js` |
| **Автор** | QA |

## Описание

Поле "Номер карты:" в форме перевода ограничивает ввод 17 цифрами, тогда как стандарт ISO/IEC 7812 для PAN-номеров банковских карт допускает 13-19 цифр; для основных международных систем (Visa, Mastercard, МИР) длина номера составляет ровно **16 цифр**, для American Express - 15. Значение 17 цифр недопустимо ни для одной популярной платёжной системы.

Дефект приводит к тому, что:

- пользователь может оформить перевод на "карту" с заведомо недопустимым номером;
- при последующей интеграции с реальным платёжным шлюзом запрос будет отклонён, а пользователь получит позднюю и неинформативную ошибку;
- маскировка `0000 0000 0000 0000` визуально намекает пользователю, что 16 цифр - корректная длина, и поведение поля противоречит подсказке.

## Шаги воспроизведения

1. Открыть `http://localhost:8000/?balance=10000&reserved=0`.
2. Кликнуть по карточке "Рубли".
3. В поле "Номер карты:" начать вводить цифры подряд: `12345678123456789` (17 знаков).
4. Посмотреть на содержимое поля.

## Ожидаемый результат

- В поле сохраняются ровно **16 цифр** (`1234 5678 1234 5678`); 17-й и последующие символы отбрасываются на этапе ввода.

## Фактический результат

- В поле сохраняются **17 цифр** (`1234 5678 1234 5678 9`).
- Форма "Сумма перевода:" отображается с 16-го символа, далее ввод не блокируется до 17 символов включительно.
- Кнопка "Перевести" доступна, в alert уходит 17-значная строка номера карты.

## Анализ кода

В обработчике `onChange` поля номера карты:

```js
let L = R.target.value.replace(/\D/g, "");
L.length > 17 && (L = L.slice(0, 17));     // <-- ограничивает 17 цифрами вместо 16
E(L);                                       // setState(card)
L = L.match(/.{1,4}/g)?.join(" ") || L;
R.target.value = L;
```

## Рекомендация

Заменить `> 17` на `> 16`:

```js
if (L.length > 16) L = L.slice(0, 16);
```

или вынести максимальную длину в константу:

```js
const CARD_MAX_DIGITS = 16;
if (L.length > CARD_MAX_DIGITS) L = L.slice(0, CARD_MAX_DIGITS);
```

Дополнительно - алгоритм Луна (контроль валидной цифры) и проверка на минимальную длину перед показом блока "Сумма перевода:".
